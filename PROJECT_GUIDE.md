# Family Pay App — מדריך הפרויקט

## מה האפליקציה עושה?

אפליקציה לניהול הוצאות אשראי משותפות במשפחה.
כל כרטיס אשראי שייך לבעל ספציפי. כשמשתמש מבצע עסקה על כרטיס של מישהו אחר — נוצר חוב לבעל הכרטיס.
בסוף כל חודש המערכת מחשבת כמה כל בן משפחה חייב לכל בעל כרטיס, על סמך העסקאות שבוצעו.

## מסכים

### מסך ראשי (Dashboard)
סקירה כללית של החודש: סך הוצאות, הוצאה אישית של בעל הכרטיס, כמה חייבים לו, ורשימת המשתמשים שעדיין לא שילמו.
**Selector:** אם יש מספר בעלי כרטיסים — אפשר לעבור בין הנקודות מבט.

### מסך עסקאות
רשימת כל העסקאות עם אפשרויות:
- חיפוש לפי שם חנות או סכום
- פילטור לפי כרטיס / לא משויך
- שיוך עסקה למשתמש
- עריכה ומחיקה של עסקה

### מסך הוספת/עריכת עסקה
טופס להוספת עסקה חדשה עם שדות: סכום, בית עסק, תאריך, הערה, קטגוריה, כרטיס, שיוך למשתמש.
אותו טופס משמש לעריכת עסקה קיימת — מתמלא אוטומטית עם הנתונים הקיימים.

### מסך משתמשים
ניהול בני המשפחה:
- הוספה, עריכה, מחיקה של משתמש
- שיוך **מספר כרטיסים** לכל משתמש (בחירה מרובה)
- ניהול כרטיסי אשראי — הוספה, מחיקה, הגדרת בעלים
- Badge "בעל כרטיס" למשתמשים שיש כרטיס על שמם

### מסך סיכום חודשי
- ניווט בין חודשים
- Selector בעל כרטיס
- טבלת התחשבנות לכל משתמש (כמה חייב, כמה עסקאות)
- כפתור "סמן כשולם" / ביטול
- "הצג דוח" — פותח מודל עם שני גרפי עוגה

### מסך ניתוח (Analytics)
פילוח גרפי עם בר-גרפים לפי קטגוריה ולפי משתמש, עם ניווט בין חודשים וסלקטור בעל כרטיס.

### מסך ייבוא (Import)
ייבוא עסקאות מקובץ XLSX של MAX:
- זיהוי אוטומטי של כרטיס לפי 4 ספרות אחרונות
- שיוך אוטומטי למשתמש אם רק משתמש אחד משויך לאותו כרטיס
- תצוגה מקדימה עם תג ירוק/צהוב לכל עסקה

## שכבת הנתונים

### טיפוסים (src/types/index.ts)

```typescript
User          — משתמש: id, שם, תפקיד, אווטאר, cardIds[] (מערך כרטיסים)
Card          — כרטיס אשראי: id, 4 ספרות אחרונות, ownerId (בעלים)
Transaction   — עסקה: id, חנות, סכום, תאריך, כרטיס, קטגוריה, משתמש, הערה
MonthlySummary — סיכום חודשי מחושב (נגזר מהעסקאות, לא נשמר)
```

### AppContext (src/context/AppContext.tsx)

| פעולה | תיאור |
|-------|--------|
| `addTransaction` | הוספת עסקה חדשה |
| `updateTransaction` | עריכת עסקה קיימת |
| `deleteTransaction` | מחיקת עסקה |
| `assignTransaction` | שיוך עסקה למשתמש |
| `addUser` | הוספת משתמש |
| `updateUser` | עריכת משתמש |
| `deleteUser` | מחיקת משתמש |
| `assignCardsToUser` | שיוך מערך כרטיסים למשתמש |
| `addCard` | הוספת כרטיס אשראי |
| `deleteCard` | מחיקת כרטיס |
| `markSettled` | סימון משתמש כ"שילם" |
| `unmarkSettled` | ביטול סימון תשלום |

`cardOwners` — נגזר אוטומטית: רשימת משתמשים שיש כרטיס על שמם.

### לוגיקת חובות (src/utils/summaryUtils.ts)

```
לכל עסקה:
  card = cards.find(c => c.id === tx.cardId)
  אם tx.assignedUserId === card.ownerId → לא חוב (בעל הכרטיס השתמש בכרטיס שלו)
  אחרת → tx.assignedUserId חייב card.ownerId את tx.amount
```

## בסיס הנתונים — Supabase

### טבלאות

**users**
| עמודה | סוג | תיאור |
|-------|-----|--------|
| id | text | מזהה ייחודי |
| name | text | שם המשתמש |
| role | text | תפקיד (משתמש / בעל החשבון וכו') |
| avatar_url | text | קישור לתמונה (אופציונלי) |
| card_ids | text[] | מערך מזהי כרטיסים שבהם המשתמש משתמש |

**cards**
| עמודה | סוג | תיאור |
|-------|-----|--------|
| id | text | מזהה ייחודי |
| last_four_digits | text | 4 ספרות אחרונות |
| owner_id | text | מזהה בעל הכרטיס |

**transactions**
| עמודה | סוג | תיאור |
|-------|-----|--------|
| id | text | מזהה ייחודי |
| store_name | text | שם בית העסק |
| amount | numeric | סכום |
| date | text | תאריך (YYYY-MM-DD) |
| card_id | text | הכרטיס שבו בוצעה העסקה |
| category | text | קטגוריה |
| assigned_user_id | text | למי שויכה (null = לא משויך) |
| is_split | boolean | האם מפוצלת |
| notes | text | הערה חופשית |

**settled_users**
| עמודה | סוג | תיאור |
|-------|-----|--------|
| user_id | text | מזהה משתמש |
| settled_date | text | תאריך התשלום |
