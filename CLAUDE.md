# Project: Shared Credit Card Expense Manager

## רקע ומטרת הפרויקט
אפליקציה לניהול הוצאות אשראי משותפות. ישנם מקרים בהם משתמשים שונים חולקים כרטיס אשראי אחד, או משתמשים בכרטיסים שונים המקושרים לאותו חשבון בנק מרכזי. 
המטרה הסופית של האפליקציה: בסוף כל חודש, המערכת תדע לחשב ולהציג במדויק כמה כסף כל משתמש (User) צריך להעביר לבעל החשבון המרכזי (Account Owner), על סמך העסקאות שבוצעו.

## גישת פיתוח ופרדיגמות
1. **Frontend-First (UI-First):** אנו מתחילים בבניית הממשק החזותי והלוגיקה הבסיסית. הפרונטאנד ייבנה על סמך קטעי קוד עיצוביים (UI) שהמשתמש יביא מכלים חיצונים (כמו גוגל Stitch).
2. **Mock Data:** בשלבים הראשונים לא יוקם מסד נתונים. הלוגיקה העסקית והממשק ייבנו על בסיס נתוני דמה (Mock Data) עם ממשקים (Interfaces) מוגדרים היטב.
3. **Clean Architecture & SOLID:** על הקוד להיות מודולרי. יש להפריד לחלוטין בין שכבת התצוגה (UI), שכבת הלוגיקה והחישובים, ושכבת הנתונים (שתתווסף בהמשך). יש לשאוף לפרדיגמות של תכנות מונחה עצמים (OOP) היכן שהן תורמות לסדר ולקריאות הקוד.
4. **טכנולוגיות גמישות:** ה-Stack הטכנולוגי (Frameworks, Database, Backend) אינו סגור מראש. על Claude להתייעץ עם המשתמש לפני כל החלטה על הוספת טכנולוגיה, ספריה או תשתית חדשה.

## חלוקת תפקידים וכללי עבודה
* **המשתמש (User):** מתפקד כארכיטקט, מנהל המוצר ואיש ה-UI. מספק את החזון, מביא את עיצובי המסכים, ומאשר החלטות טכנולוגיות.
* **קלוד (Claude Code):** מתפקד כמתכנת הראשי (Lead Developer) והיועץ הטכנולוגי.
  - באחריותך לכתוב קוד נקי ולנהל את קבצי הפרויקט.
  - באחריותך להציע ולהתייעץ לגבי הכלים ושפות התכנות הנכונות ביותר לכל שלב.
  - יש לך רשות להריץ פקודות בטרמינל (התקנות, הרצת שרת מקומי וכו') כדי לבדוק את הקוד.
  - באחריותך לבצע דיבוג עצמאי: קרא שגיאות מהטרמינל ותקן את הקוד בהתאם.

## מסכים שעוצבו (Google Stitch)
המסכים עוצבו בכלי Google Stitch ושמורים בתיקיית `screens/`:

| קובץ | מסך | תיאור |
|------|-----|--------|
| `screens/dashboard.html` | ראשי | סקירה כללית — סיכום הוצאות חודשי, יתרה, חייבים לך, סטטוס משתמשים |
| `screens/transactions.html` | עסקאות | רשימת עסקאות עם חיפוש, פילטרים, ושיוך/פיצול לפי משתמש |
| `screens/users.html` | משתמשים | ניהול משתמשים וכרטיסים — שיוך כרטיס לכל משתמש |
| `screens/summary.html` | סיכום חודשי | סטטוס התחשבנות מול כל משתמש — כמה חייב, מה שולם, ייצוא דוח |
| `screens/add-transaction.html` | הוספת עסקה | טופס הוספת עסקה חדשה עם שיוך משתמש ופיצול |

### עיצוב ו-Design System
- **צבעים:** Material Design 3 color tokens (primary, surface, tertiary וכו')
- **פונטים:** Manrope (כותרות), Inter (גוף)
- **אייקונים:** Material Symbols Outlined
- **CSS Framework:** Tailwind CSS (נטען מ-CDN)
- **כיוון:** RTL, עברית

## Stack טכנולוגי
- **Vite** — כלי הבנייה ושרת הפיתוח המקומי
- **React** — ספריית ממשק המשתמש (רכיבים דינמיים)
- **TypeScript** — שפת הפרויקט (אכיפת מבני נתונים ובדיקות סוג)
- **Tailwind CSS v4** — מערכת העיצוב (זהה לזו שבמסכי Stitch)

### פקודות שימושיות

```bash
npm run dev    # הרצת שרת פיתוח מקומי
npm run build  # בנייה לייצור

```
